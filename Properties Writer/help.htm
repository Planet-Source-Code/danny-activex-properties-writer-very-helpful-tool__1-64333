Introduction

Properties Writer was designed as a tool for programmers who often write ActiveX controls. It takes the time and hassle out of typing properties into your code manually. With an easy-to-use interface and fast code generation, Properties Writer is a great tool for all Visual Basic programmers.

How it works

First, you enter your property information into the form, such as: property name, data type, container variable, etc. (each field will be discussed soon). After all information is provided, you can click the Add button to add the property to the list.

Once you have all of your properties added to the list, it is recommended you save the session by clicking the File -> Save menu item. This will save a list of all properties so you can modify them at a later time.

After your property list is saved, you can generate the Visual Basic code by clicking the Generate -> Generate Code menu item. A new window will appear displaying the code to copy and paste into your user control project.

Properties Writer generates code for the following:

  • Container variable declarations
  • Property get and property set/let routines
  • UserControl_InitProperties() event
  • UserControl_ReadProperties() event
  • UserControl_WriteProperties() event

Changing program settings

Properties Writer comes with several different options to customize the way it behaves. The first category in the list is Program Behavior.

Below, is a brief description of what each setting does and it's default value.

Save window position when closing - If this feature is enabled, Properties Writer will save its last used position on-screen, so next time you use it, it will appear exactly where it was before. You can reset this value by clicking the corresponding Reset button. This feature is enabled by default.

Save list column sizes when closing - If this feature is enabled, Properties Writer will save the widths of the property list's columns so you don't have to resize them every time you use the program. You can reset this value by clicking the corresponding Reset button. This feature is enabled by default.

Save property clear options when closing - If this feature is enabled, Properties Writer will save the Clear options on the main form so you don't have to change them every time you use the program. This feature is enabled by default.

The next category in the list is Code Generator.

Below, is a brief description of what each setting does and it's default value.

Add comments to code - If this feature is enabled, Properties Writer will add brief comments to the generated code. Comments are placed:

  • Before property container variable declarations
  • Before property get, set/let routines
  • Before the UserControl_InitProperties() event
  • Before the UserControl_ReadProperties() event
  • Before the UserControl_WriteProperties() event

This feature is disabled by default.

Indent code between sub routine/end sub - If this feature is enabled, Properties Writer will indent the code within a sub routine. For example, the code will look like the following:

Private Sub UserControl_InitProperties()
    Dim Variable As String
   
    'Insert code here...
    Variable = "Code here."
End Sub

This feature is enabled by default.

Indent code between with/end with statements -  If this feature is enabled, Properties Writer will indent the code between a With/End With statement. For example, the code will look like the following:

Private Sub UserControl_InitProperties()
    With Object
        .Property1 = "Value"
        .Method
    End With
End Sub

This feature is enabled by default.

Indent using real tabs - If this feature is enabled, Properties Writer will use real tab characters (vbTab).

Indent width (spaces) - If the indent using real tabs feature is disabled, you can specify the number of spaces to use for each tab. For example, if you entered a value of 2 spaces, then the code will look like the following:

Private Sub UserControl_InitProperties()
  With Object
    .Property1 = "Value"
    .Method
  End With
End Sub

This feature is disabled by default.

Entering basic information

When adding a property to the list, you must first enter its information. If you are not familiar with how properties work in an ActiveX control, then it is recommended that you learn that first.

Below is a brief description of what each field is for:

Property name - This is the name of the property. For example, Font, BackColor, Caption, etc. This is the name that will appear in your ActiveX control's property list.

Data type - This is the type of data your property holds. For example, String, Boolean, Integer, StdFont, etc. You can also enter a custom data type that you define in your user control, for example:

Public Enum CONTROL_STYLE
    Standard = 0
    [XP Blue] = 1
    [XP Silver] = 2
End Enum

Using the above example, we would enter CONTROL_STYLE in the data type field.

Container - This is the name of the container variable that holds the value of the property. For example:

Private strCaption As String 'This is the container for the Caption property.

Public Property Get Caption() As String
    Let Caption = strCaption
End Property

Using the above example, we would enter strCaption in the container field.

Default value - This is the default value of your property. This will be the property's value when the user first adds your ActiveX control to their project. For example:

Private Sub UserControl_InitProperties()
    Let Caption = Ambient.DisplayName
End Sub

Using the above example, we would enter Ambient.DisplayName into default value field. Note that if the default value is a string, make sure you enter quotes into the default value field. For example:

Private Sub UserControl_InitProperties()
    Let Caption = "MyControl1"
End Sub

Using the above example, we would enter "MyControl1" into the default value field (with quotes).

Clear options - To the very right of each field is a checkbox. If selected, this field will be cleared after you add the property to the list. If left blank, the value you used on the last property will remain.

Property is object - Enable this feature if the property is an object. For example, a Font (StdFont), Picture (StdPicture), etc. This tells the program to use Set instead of Let in the code.

Property is read-only - Enable this feature if the property is read-only. If a property is read-only, it cannot be changed by the user. For example:

Public Property Get hWnd() As Long
    hWnd = UserControl.hWnd
End Property

If the property is read-only, Properties Writer will not generate code for the:

  • Property container variable declarations
  • UserControl_InitProperties() event
  • UserControl_ReadProperties() event
  • UserControl_WriteProperties() event

Adding a property

After you have entered all of the property's information, click the Add button at the bottom of the form to add it to the list.

Back to top

Removing a property

If you wish to remove a property, simply select it in the list and click the Remove button.

Back to top

Editing a property

If you wish to edit a property, simply select it in the list and click the Edit button. The information will be displayed in the fields and the Edit button will turn into an Update button. After you're finished editing the property's information, click the Update button and the property will be updated.

Back to top

Generating code

After all of the properties have been added to the list, it is recommended that you first save the file by clicking the File -> Save menu item. Once the file is saved, you can generate the code by clicking the Generate -> Generate Code menu item or simply by pressing CTRL + G.

A new window will appear containing all of the code that was generated. Simply copy and paste the code into your ActiveX control project and you're done!
